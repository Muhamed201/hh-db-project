# Проектирование и оптимизация базы данных

В данном репозитории представлена реализация базы данных для сервиса поиска работы. Проект выполнен в рамках учебного задания и разбит на 6 этапов.

## Состав проекта

1. **01_schema.sql** — Проектирование структуры БД (таблицы: вакансии, резюме, отклики, специализации, регионы).
2. **02_seed_data.sql** — Скрипт генерации тестовых данных (10 000 вакансий, 100 000 резюме и 150 000 откликов).
3. **03_avg_compensation.sql** — Аналитический запрос: расчет средних зарплат (от, до, среднее) в разрезе регионов.
4. **04_peak_activity.sql** — Поиск месяцев с наибольшей активностью по публикациям вакансий и резюме.
5. **05_popular_vacancies.sql** — Выборка вакансий, получивших более 5 откликов в первую неделю после публикации.
6. **06_indexes.sql** — Создание индексов (B-Tree) для оптимизации поиска, группировки и JOIN-операций.

## Технологии
* **СУБД:** PostgreSQL

## Особенности реализации
* Использована логика случайного распределения данных для имитации реальной нагрузки.
* Применены индексы на внешние ключи и поля дат для ускорения сложных аналитических запросов.
* Структура таблиц нормализована для исключения дублирования данных (справочники регионов и специализаций).
